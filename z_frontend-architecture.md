## üîÑ Data Flow Architecture

### File Processing Workflow
1. **Upload**: `AIFileUpload` ‚Üí Supabase Storage ‚Üí Database record
2. **Detection**: File type detection ‚Üí Processing method selection (includes CIM detection)
3. **Processing**: `aiApi.ts` ‚Üí AI Server ‚Üí Structured results
4. **Progress Tracking**: Real-time status updates via `CIMProcessingProgress`
5. **Storage**: Results ‚Üí Supabase tables ‚Üí Real-time UI updates
6. **Display**: Component re-renders ‚Üí User sees analysis in dedicated views

### CIM Processing Flow
```typescript
// Enhanced CIM processing with progress tracking
const processCIMWorkflow = async (file: File, dealId: string) => {
  // 1. Validation (0-25%)
  const validation = validateCIMFile(file);
  
  // 2. Analysis (25-75%)
  const result = await processCIM(file, dealId, documentId);
  
  // 3. Storage (75-95%)
  await storeCIMAnalysis(dealId, documentId, result.parsed_analysis);
  
  // 4. Complete (100%)
  // Update UI with CIMAnalysisDisplay component
};
```
